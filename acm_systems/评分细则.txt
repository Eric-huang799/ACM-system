========================================================================
                        ACM程序设计竞赛评分规则
========================================================================
一、比赛形式
  参赛单位：个人或团队（3人/队）
  比赛时长：4C5 小时
  题目数量：6C10 道
  支持语言：C 
  提交方式：通过在线评测系统（OJ）
二、评测结果说明
+------------------+------------------------------+------------------+
| 状态缩写         | 含义                          | 是否计入罚时      |
+------------------+------------------------------+------------------+
| AC               | 正确通过                      | 是                |
| WA / TLE / MLE   | 答案错 / 超时 / 内存超限       | 是（若后续AC）    |
| RE / PE          | 运行错误 / 格式错误            | 是（若后续AC）    |
| CE               | 编译错误                      | 否                |
+------------------+------------------------------+------------------+
三、排名规则（优先级从高到低）
  1. 正确解题数（越多越靠前）
  2. 总罚时（越少越靠前）
  3. 队伍ID字典序（平局时使用）
四、罚时计算公式
  某题罚时 = 首次AC时间（分钟） + 20 ×（该题AC前的错误提交次数）
  总罚时 = 所有AC题目的罚时之和
  （注：CE不计罚时；未AC题目不计任何时间）
五、奖项设置
+----------+--------------+----------------------------------+
| 奖项      | 比例         | 说明                              |
+----------+--------------+----------------------------------+
| 一等奖    | ≤20%         | 解题最多且罚时最低               |
| 二等奖    | ≤30%         |                                  |
| 三等奖    | ≤50%         |                                  |
| 总获奖率  | ≤参赛总数35% | 符合教育部学科竞赛规范           |
+----------+--------------+----------------------------------+
六、注意事项
  - 比赛最后1小时可启用“封榜”
  - 禁止抄袭、代写、联网，违者取消资格
  - 气球奖励：每AC一题发对应颜色气球
========================================================================
